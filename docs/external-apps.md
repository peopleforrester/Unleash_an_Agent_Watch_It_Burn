# External apps (extracted from this repo)

The four Railway-hosted apps that used to live under `railway/` and `lab-distribution/`
were split into their own repos so they can be developed independently. They are **no
longer in this repo** and must not be re-added here. Each is its own private GitHub repo
under `peopleforrester`, deployed by Railway from its `main` branch.

| App | Old path (removed) | Sibling repo | Live URL |
|---|---|---|---|
| Walkthrough deck | `railway/walkthrough/` | `peopleforrester/walkthrough-agenticburn` | https://walkthrough.agenticburn.com |
| Rounds + Challenges deck | `railway/rounds-walkthrough/` | `peopleforrester/rounds-agenticburn` | https://rounds.agenticburn.com |
| Apex / wildcard router (Caddy) | `railway/apex/` | `peopleforrester/apex-agenticburn` | https://agenticburn.com (apex + `*.agenticburn.com`) |
| Provisioning distributor (Flask) | `lab-distribution/` | `peopleforrester/provisioning-agenticburn` | https://provisioning.agenticburn.com |

On a workstation they are expected as siblings of this repo (e.g. `~/repos/events/<app>-agenticburn`).

## The one remaining coupling: fleet -> provisioning

`infra/terraform/fleet/fleet.sh` still runs the provisioning **harvester scripts** during
`ingest` (`harvest_cluster_access.sh`, `generate_attendee_aws.py`, `push_vtt_aws_creds.sh`,
and `merge_pool.py` / `distribute_datadog_keys.py`). Those scripts moved with the app into
`provisioning-agenticburn/scripts/`. `fleet.sh` finds them via **`WIB_PROVISION_DIR`**
(default `${REPO_ROOT}/../provisioning-agenticburn`); override the env var if the sibling
checkout lives elsewhere. The runtime data population is otherwise decoupled: the real
`pool.csv` is dropped into the provisioning app at deploy (gitignored), and the app seeds
its SQLite from it on a `/data` volume.

## The apex router data handoff

`apex-agenticburn`'s `routes.map` (Host -> per-cluster LB hostname) is regenerated by the
fleet tooling here; the new `routes.map` is committed into `apex-agenticburn` and a push to
its `main` redeploys the router. No code dependency, only the data handoff.

## Conventions (carried into each sibling repo's CLAUDE.md)

Railpack never Nixpacks; Railway deploys from `main` via the GitHub integration (not
`railway up`); custom domains are bound to the Railway *service* (never recreate the
service); commits carry no AI attribution and the pre-commit hook also rejects the literal
word "LLM".
