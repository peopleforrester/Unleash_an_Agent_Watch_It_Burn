# Changelog

All notable changes to this project will be documented in this file.
Format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Added
- (2026-06-22) Created the observability meta-PRD (GitHub issue #7, `prds/7-observability-meta.md`): 8 self-contained milestones that each produce a research synthesis or a child PRD covering one area of the observability strategy for the Watch It Burn workshop. Each milestone is executable by a cold AI instance — it lists its own reads, design decisions, and child-PRD creation steps. Decisions are documented in each child PRD's Decision Log rather than separate planning docs.
- (2026-06-22) Created `prds/` directory to hold PRD files; added a local `.gitignore` override since the global gitignore excludes `prds/`.
- (2026-06-22) Created `docs/ROADMAP.md` with forward-looking implementation order for observability PRDs.

### Changed
- (2026-06-23) Created three research spike issues (#9, #10, #11, #12) to gate Milestone 2, 3, and 5 design conversations: #9 covers Datadog LLM Observability OTLP ingestion, #10 covers Python AI-layer instrumentation approach toward OTel GenAI semantic conventions, #11 covers per-component telemetry synthesis for all 13 stack components, and #12 covers how to capture before/after prompt text in guard-proxy traces (the re-leak trap beat). Updated the meta-PRD to reference each spike in its respective milestone step and clarified that wire-or-skip decisions belong to the M5 design conversation, not the research deliverable.
- (2026-06-23) Added notes to research spike issues #9 and #12 that "Datadog LLM Observability" may now be part of a broader "Agent Observability" platform; both spikes should verify the current product name. Issue #12 also now covers whether Datadog's built-in Sensitive Data Scanner applies to manually instrumented spans, or whether Collector-side symmetric redaction is still the right path for the re-leak trap. Fixed a stale research file reference in Milestone 7 (community-dashboard survey is now question 5 inside the issue #11 per-component synthesis, not a separate research/28 file). Recorded two new meta-PRD decisions: wire-or-skip moved to the Milestone 5 design conversation, and guard-proxy before/after sanitization confirmed as its own research spike (issue #12).
- (2026-06-23) Corrected the meta-PRD so that Milestone 2 and 3 design conversations reference pre-completed research spikes (issues #9, #10, #12) rather than instructing the implementing AI to run /research itself. The implementing AI now reads the pre-completed output and stops if the issue is not yet complete. Downstream "Done when" checklists and child-PRD step references updated to match (decisions 2-7 and 2-4 respectively, since step 1 is now a reading prerequisite, not a numbered design decision).
- (2026-06-22) Updated observability planning docs and research/23 to reflect that the TypeScript rewrite of guard-proxy is not happening — all Python apps stay Python, spiny-orb instrumentation is off the table, Decision 4 in research/23 closed as superseded.
- (2026-06-23) Completed the Milestone 1 design conversation for the observability plan: locked all cross-cutting decisions (OTel Collector pipeline shape, Unified Service Tagging vocabulary per component, MVP Datadog account model), produced the MVP child PRD (#13) with implementation steps for wiring the Datadog trial account end-to-end, and created `docs/observability-priorities.md` as a living must-have/nice-to-have reference that every future milestone design conversation reads first.
- (2026-06-23) Clarified Weaver's role in the observability plan: it stays in Milestone 2 (not Milestone 1) because gen_ai.* spans don't exist until the migration happens; the OTel community semconv registry provides the upstream gen_ai.* attribute definitions as a Weaver dependency, so no vocab needs to be pre-defined earlier. Fixed two stale Decision Log entries that contradicted this. Also improved the meta-PRD so that "clear context" instructions correctly tell the user to start a new session rather than asking the AI to clear its own context.
