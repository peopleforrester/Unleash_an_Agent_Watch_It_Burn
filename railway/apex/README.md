# agenticburn.com apex + wildcard router (`watch-it-burn-apex`)

One Railway service that does two jobs:

1. **Landing page** at `agenticburn.com` / `www` (static, in `site/`).
2. **Wildcard router** for `*.agenticburn.com` — proxies each attendee/facilitator host to its cluster's
   public LoadBalancer, from `routes.map`:
   - `a-<id>.agenticburn.com` → that attendee's Cluster 3
   - `burn` / `wall` / `haiku` / `sonnet` / `opus` → the facilitator clusters

Railway terminates TLS at its edge (one **wildcard cert**, see below), so Caddy here just serves plain
HTTP on `$PORT` and routes by `Host`. This keeps everything on Railway — **netcup is not in the path.**

## Why this is on Railway, not netcup

Railway supports **wildcard custom domains** and issues the wildcard cert by delegating the
`_acme-challenge` record to its own side — so the Namecheap IP-allowlist constraint (which only matters
for DNS-01 run by us) does not apply. No whitelisted IP, no DNS-01 work for us, no netcup.

## Files

- `Caddyfile` — apex → landing; other hosts → `routes.map` upstream; unknown host → `site/unknown.html` (404).
- `routes.map` — `host  http://<cluster-lb>` table. **Generated by the provisioner**; the committed copy
  is a documented placeholder (no real LB hostnames — clusters are per-event and torn down to $0).
- `site/` — landing + unknown-host pages (public-safe; no creds/URLs/internal detail).
- `Dockerfile` / `railway.json` — Caddy image, `DOCKERFILE` builder.

## Deploy (Railway)

```bash
cd railway/apex
railway link --project <id> --service watch-it-burn-apex --environment production   # one-time
railway up --ci --service watch-it-burn-apex
railway variables --set PORT=8080         # Caddy listens on $PORT; set if the edge needs a fixed port
```

## Custom domains (the careful part — needs Michael's go for the DNS write)

Add the domains to the service, then set the records Railway prints at Namecheap (read-then-merge so the
existing `walkthrough` + parking records survive — see `~/.claude/rules/tools/namecheap-api.md`):

```bash
railway domain agenticburn.com --service watch-it-burn-apex      # apex landing
railway domain '*.agenticburn.com' --service watch-it-burn-apex  # wildcard router
```

Wildcard yields **two CNAMEs (wildcard + `_acme-challenge`) + one ownership TXT** — all three are
required or the wildcard cert won't issue. `walkthrough.agenticburn.com` is a specific record and keeps
winning over the wildcard, so the deck is unaffected.

## Updating routes

Regenerate `routes.map` from the fleet/pool output and redeploy (`railway up`). Repointing a Cluster-1
spare = edit its one line + redeploy. Routes only resolve once the target clusters are provisioned.
