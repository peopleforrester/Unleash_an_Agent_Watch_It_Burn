# Watch It Burn: lab/key distribution

A small Flask app that hands out one pre-provisioned EKS cluster per attendee from a pool, claimed
atomically by email. Adapted from the KCD Texas 2026 distributor (portfolio/lab-provisionin-website).
Pulled in as the starting point for Watch It Burn provisioning; the code is here, the real pool and
attendee data are NOT (see .gitignore).

## How it works

- Attendee opens the URL (QR at the door), enters their email, and is assigned one cluster from
  `pool.csv` atomically (idempotent by email; re-entering the same email returns the same cluster).
- The success page shows the AWS credentials + the setup commands. Optional Resend email delivery.

## What is here vs not

- HERE (code): `app.py`, `templates/`, `static/`, `assets/`, `Procfile`, `railway.json`, `pyproject.toml`,
  `test_app.py`, and a PLACEHOLDER `pool.csv` (AKIAEXAMPLE rows, watch-it-burn-attendee naming, us-west-2).
- NOT here (gitignored): `pool.db` (claims), `data/` (real attendee emails), the real credential pool,
  any `.env`. The real pool is generated by the provisioning project and dropped in at deploy time.

## Adapt-before-use (verify-at-build)

- Confirm the cluster naming (`watch-it-burn-attendee-<id>`), region (us-west-2), and the URL scheme
  on agenticburn.com match the provisioning output (infra/dns/, infra/attendee-cluster/).
- Set `ADMIN_TOKEN` and (optional) `RESEND_API_KEY` via env, never hardcoded.
- The KodeKloud `/browser` path is inherited from the source; remove it if Watch It Burn is EKS-only.
- This is a 2-hour lab tool: provision the pool before doors, tear it down after.

## Run locally

```bash
cd lab-distribution
uv run python app.py   # or: flask run ; ADMIN_TOKEN required
```
