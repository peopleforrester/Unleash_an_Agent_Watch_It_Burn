3 · Optional: connect from your own laptop (local kubectl)

Skip this whole section if you are working in the in-browser console above. It already has a terminal. You only need the steps below to run kubectl from your own machine.

a. Install the tools

You need two command-line tools: the AWS CLI v2 and kubectl. Open the tab for your OS.

macOS (Homebrew, or the official installer) 
# AWS CLI v2  (or download https://awscli.amazonaws.com/AWSCLIV2.pkg and run it)
brew install awscli
# kubectl
brew install kubectl
# verify both are installed
aws --version
kubectl version --client
Windows (PowerShell + winget) 
# AWS CLI v2  (or download https://awscli.amazonaws.com/AWSCLIV2.msi and run it)
winget install -e --id Amazon.AWSCLI
# kubectl
winget install -e --id Kubernetes.kubectl
# close and reopen PowerShell, then verify
aws --version
kubectl version --client
Linux (x86_64) 
# AWS CLI v2
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o awscliv2.zip
unzip awscliv2.zip && sudo ./aws/install
# kubectl
curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
# verify both are installed
aws --version
kubectl version --client

b. Your AWS keys

Access key {{ access_key }}
Secret key {{ secret_key }}

c. Configure the AWS CLI

Run aws configure. It asks four questions, one at a time. Answer them like this (paste the keys from step b, type the region and format exactly as shown): 

$ aws configure
AWS Access Key ID [None]:      <paste your access key>
AWS Secret Access Key [None]:  <paste your secret key>
Default region name [None]:    {{ region }}
Default output format [None]:  json

d. Connect to your cluster

aws eks update-kubeconfig --name {{ cluster_name }} --region {{ region }}
kubectl get nodes                              # confirm your cluster is up
Most of the workshop is in the console. Drive the agent from the chat tab and use the terminal tab for kubectl. Everything in section 3 is only for the optional local-kubectl path.
